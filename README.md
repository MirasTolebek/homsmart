# 🌡️ Умная автоматика циркуляционного насоса (ESP32 + Telegram Bot)
Открытый проект IoT-контроллера для автоматического управления циркуляционным насосом системы отопления (твердотопливной печи). Система защищает котел от закипания и экономит ресурс насоса, предотвращая его работу вхолостую на остывшей печи.
Оснащена удобным управлением через Telegram-бота с использованием динамических Inline-кнопок (без спама в истории чата).
## ✨ Главные возможности
 * **Полная автоматизация (Гистерезис):** Насос включается при растопке (заданный порог нагрева) и выключается при прогорании печи (порог остывания).
 * **Telegram-интерфейс:** Мониторинг температуры и управление системой прямо из мессенджера из любой точки мира.
 * **Inline-клавиатура:** Статус системы обновляется "на лету" в одном сообщении.
 * **Настройка "по воздуху":** Температурные пороги изменяются простыми текстовыми командами (например, /нагрев 60).
 * **Энергонезависимая память (EEPROM):** Настройки сохраняются во Flash-памяти и не сбрасываются при отключении электричества.
 * **Отказоустойчивость (Fail-Safe):** Non-blocking архитектура (без delay()). При падении Wi-Fi плата переходит в глубокий автономный режим и продолжает надежно управлять насосом.
## 🛠️ Аппаратная часть (Hardware)
 * **Микроконтроллер:** ESP32 (протестировано на версии Super Mini).
 * **Датчик температуры:** DS18B20 (водонепроницаемый, в стальной гильзе).
 * **Обвязка датчика:** Подтягивающий резистор 4.7 кОм.
 * **Реле:** Обычный 5V модуль реле с управлением по низкому уровню (Low Level Trigger).
### Схема подключения (Pinout)

| Компонент | Вывод | Пин ESP32 | Примечание |
| :--- | :--- | :--- | :--- |
| **DS18B20** | VCC | 3.3V | Питание датчика |
|  | GND | GND | Земля |
|  | Data | GPIO 4 | Обязательна подтяжка резистором 4.7 кОм к 3.3V |
| **Модуль Реле** | VCC (+) | 5V | Питание катушки реле от 5V! |
|  | GND (-) | GND | Земля |
|  | IN (Сигнал) | GPIO 5 | Управляется логикой 3.3V от ESP |

*Силовая часть (220V):* В разрыв фазового провода насоса подключаются контакты реле **COM** (общий) и **NO** (нормально разомкнутый).
## 📦 Программные зависимости
Проект требует установки следующих библиотек в Arduino IDE.
> ⚠️ **ВНИМАНИЕ:** Строго соблюдайте версии библиотек!
> 
 * UniversalTelegramBot (версия **1.3.0**)
 * ArduinoJson (строго версия ветки **6.x**, например **6.21.5**). *Версия 7.x не поддерживается библиотекой бота!*
 * OneWire (версия **2.3.8** или новее)
 * DallasTemperature (версия **4.0.6** или новее)
## 🚀 Установка и запуск
 1. Склонируйте репозиторий или скачайте код.
 2. Создайте файл secrets.h в одной папке со скетчем .ino. Это защитит ваши пароли от случайной публикации.
 3. Добавьте в secrets.h свои данные:
   ```cpp
   // Файл secrets.h
   const char* ssid     = "ВАШ_WIFI_SSID"; 
   const char* password = "ВАШ_WIFI_ПАРОЛЬ";  
   
   #define BOTtoken "ТОКЕН_ВАШЕГО_БОТА_ОТ_BotFather"
   #define CHAT_ID  "ВАШ_TELEGRAM_CHAT_ID"
   
   ```
 4. Выберите в Arduino IDE вашу плату, скомпилируйте и загрузите код.
## 📱 Использование Telegram-бота
При старте бот пришлет панель управления.
 * 🔄 **Обновить статус** — запрашивает актуальную температуру.
 * 🟢 **Включить / ⚪ Выключить** — ручное управление насосом.
 * 🤖 **Авторежим** — возврат к автоматическому управлению.
 * ⚙️ **Настройки** — управление Wi-Fi и уведомлениями.
**Установка порогов (отправьте текстовым сообщением):**
 * /нагрев 55.5 — насос включится при 55.5°C.
 * /остывание 45.0 — насос выключится при 45.0°C.
